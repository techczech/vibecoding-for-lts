# Source Outline

This public-safe source outline is adapted from a private draft note. It keeps
the session structure and removes local vault provenance from committed files.

## Core Frame

- AI work is moving from chatbot use towards agent-supported tool making and
  tool use.
- Learning technologists need to understand both model capability and the
  harness that connects a model to local files, browsers, APIs, and course
  platforms.
- The workshop should give participants direct experience with web-based tool
  making, desktop coding agents, Canvas workflows, and single-file HTML course
  activities.

## Session Flow

1. Experience tool making with Google AI Studio.
2. Experience tool use with Codex on a local folder.
3. Manage a Canvas sandbox course with Codex.
4. Turn a web activity into a single HTML file and embed it in Canvas.
5. Set up repeatable Codex practices.

## Factual Freshness Notes

- Current model and harness rankings must be refreshed close to delivery.
- Product names and capabilities should be checked before publication.
- Canvas API steps should be tested in a sandbox course before the live session.
